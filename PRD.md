## 1. 제품 개요

### 1.1 제품명
**법률자문 FAQ 검색 시스템** — 재단법인 씨젠의료재단 준법감사팀 내부 시스템

### 1.2 한 줄 설명
사용자가 키워드를 입력하면 AI가 씨젠의료재단의 실제 법률자문 사례에서 관련 Q&A를 찾아 FAQ 형식으로 제공하는 사내 검색 웹 시스템

### 1.3 기술 스택

| 구성 요소 | 기술 |
|---|---|
| 백엔드 서버 | Node.js + Express |
| AI 엔진 | Google Gemini API (`gemini-3.1-flash-lite`) |
| 프론트엔드 | Vanilla HTML / CSS / JavaScript (단일 페이지) |
| 데이터 소스 | Excel 파일 (`.xlsx`) — `t_feed(본문내용)` 시트 |
| Excel 파싱 | xlsx (SheetJS) v0.18 |
| 환경 변수 | dotenv |
| 호스팅 | Vercel (서버리스, `@vercel/node`) |
| 네트워크 | 사내 Zscaler 환경 대응 (`zscaler-ca.pem`) |

## 2. 사용자 페르소나

### 페르소나 A — 실무 부서 담당자 (주 사용자)
- **역할**: 씨젠의료재단 전 부서 소속 업무 담당자
- **목적**: 계약·임대차·노동·개인정보 등 법률 관련 업무 처리 전 선례 확인
- **특성**: 법률 전문 지식이 낮으며, 준법감사팀에 질문하기 전에 스스로 참고할 정보 필요
- **주요 불편사항**: 유사한 사례가 있는지 찾을 방법이 없어 매번 준법감사팀에 새 요청을 넣어야 함

### 페르소나 B — 준법감사팀 담당자
- **역할**: 시스템 운영 및 데이터 관리, 검색 품질 모니터링
- **목적**: 반복 문의 감소, 자문 이력 지식 자산화, 데이터 최신화
- **특성**: 법률 전문 지식 보유, 시스템 관리 권한 보유

## 4. 기능 요구사항

### 4.1 화면 구성

단일 페이지(SPA) 구조로, 헤더·검색 영역·결과 영역 세 섹션으로 구성된다.

| 영역 | 구성 요소 | 설명 |
|---|---|---|
| 헤더 | 시스템명, 부제, DB 배지 | 로드된 자문 건수 실시간 표시 (`/api/stats` 호출) |
| 검색 영역 | 검색 입력창, 검색 버튼, 추천 키워드 태그 | 텍스트 입력 또는 태그 클릭으로 검색 실행 |
| 결과 영역 | 초기 안내 → 로딩 스피너 → FAQ 카드 목록 또는 오류 메시지 | 검색 상태에 따라 동적 전환 |

---

### 4.2 기능 상세

#### F-01. 키워드 검색

**입력**
- 사용자가 검색창에 자연어 키워드 입력 (예: `임대차 계약`, `개인정보 제공`, `징계`)
- Enter 키 또는 [검색] 버튼으로 실행

**처리 흐름**
1. 프론트엔드에서 `POST /api/search` 호출 (`{ query: string }`)
2. 서버에서 키워드 사전 필터(keywordFilter) 실행 → 최대 15건의 후보 자문 사례 추출
3. Gemini AI가 후보 사례와 검색어를 비교·분석하여 관련성 높은 1~5건 선별
4. JSON 응답을 프론트엔드에 반환 → FAQ 카드 렌더링

**출력**
- 결과 요약 문장 (총 N건 관련 사례, 한 줄 요약)
- FAQ 카드 목록 (최대 5건)

**예외 처리**
- 빈 검색어: 검색 미실행 (클라이언트 가드)
- DB 미로드 상태: `503 FAQ 데이터베이스가 로드되지 않았습니다` 오류 메시지 표시
- AI 응답 파싱 실패: `422` 응답, 오류 메시지 표시
- 관련 사례 없음: "관련 자문 사례를 찾지 못했습니다" 안내 화면 표시

---

#### F-02. 추천 키워드 태그

검색창 하단에 주요 법률 분야를 태그로 제공한다. 태그 클릭 시 해당 키워드로 검색이 즉시 실행된다.

| 태그 | 태그 | 태그 | 태그 | 태그 |
|---|---|---|---|---|
| 임대차 | 개인정보 | 계약 해지 | 징계 해고 | 리베이트 |
| 전세권 | 개인정보 제공 | 김영란법 | 상표 등록 | 근저당 |

---

#### F-03. FAQ 카드 표시

검색 결과 각 항목은 카드 형식으로 표시된다.

| 요소 | 내용 |
|---|---|
| 카테고리 배지 | 계약검토 / 임대차 / 개인정보 / 노동·징계 / 공정거래 / 의료법 / 상표·IP / 기타 |
| 요청 부서 | 자문을 요청한 부서명 |
| 자문 완료일 | 검토 완료 날짜 |
| Q. (질문) | AI가 자문 내용을 핵심 질문으로 재구성한 한 문장 |
| A. (답변) | 검토의견을 바탕으로 한 명확한 답변 (3~6문장) |

---

#### F-04. DB 상태 배지

헤더 우측 배지가 페이지 로드 시 `/api/stats`를 호출하여 현재 데이터 상태를 표시한다.

| 상태 | 표시 | 배경색 |
|---|---|---|
| 정상 로드 | `자문 사례 N건` | 반투명 흰색 |
| DB 오류 | `DB 오류` | 반투명 빨간색 |
| 연결 실패 | `DB 연결 실패` | 반투명 빨간색 |

---

#### F-05. 로딩 상태 표시

검색 실행 중에는 다음 UI가 표시된다.
- 검색 버튼 비활성화 및 텍스트 변경 (`검색` → `검색 중...`)
- 결과 영역에 스피너 애니메이션 + `Gemini AI가 관련 자문 사례를 분석 중입니다...` 문구 표시

### 4.3 서버 데이터 처리

#### 데이터 소스 (Excel)

| 컬럼명 | 설명 | 필터 조건 |
|---|---|---|
| `REPLY_YN` | 댓글 여부 | `'N'`인 건만 (본문만) |
| `FEED_STATE` | 자문 상태 | `'REVIEW_COMPLETED'`인 건만 |
| `REVIEW_CONTENTS` | 검토 의견 | 10자 초과인 건만 |
| `TITLE` | 자문 제목 | 검색 대상 |
| `USER_NAME` | 요청자 이름 | 참고용 |
| `ORG_NAME` | 요청 부서 | FAQ 카드 표시 |
| `CONTRACT_PARTNER` | 계약 상대방 | 검색 대상 |
| `REQUEST_CONTENTS` | 요청 내용 | 검색 대상 (최대 400자 AI 전달) |
| `REVIEW_CONTENTS` | 검토 의견 | 검색 대상 및 답변 생성 기반 (최대 600자 AI 전달) |
| `REQUEST_DATE` | 요청일 | FAQ 카드 표시 |
| `REVIEW_COMPLETE_DATE` | 검토 완료일 | FAQ 카드 표시 (우선 표시) |

#### 키워드 사전 필터 (keywordFilter)

1. 검색어를 공백 기준으로 토크나이징
2. 각 토큰에 대해 제목·요청내용·검토의견·계약상대방에서 빈도 계산
3. 빈도 합산 점수 상위 15건 선택
4. 관련 사례가 3건 미만이면 나머지를 전체에서 보충하여 최소 10건 유지

### 4.4 AI 프롬프트 설계 원칙

```
역할: 씨젠의료재단 준법감사팀의 법률자문 FAQ 검색 시스템
목적: 사용자 검색어와 가장 관련성 높은 실제 자문 사례를 찾아 FAQ 형식으로 정리
선별 기준: 검색어와 자문 내용의 의미론적 관련성 (1~5건)
카테고리: 계약검토 | 임대차 | 개인정보 | 노동·징계 | 공정거래 | 의료법 | 상표·IP | 기타
```

**반환 JSON 구조**
```json
{
  "totalFound": 관련 사례 총 건수(숫자),
  "summary": "검색 결과 한 줄 요약",
  "faqs": [
    {
      "question": "핵심 질문을 한 문장으로",
      "answer": "검토의견을 바탕으로 한 명확한 답변 (3~6문장)",
      "category": "계약검토|임대차|개인정보|노동·징계|공정거래|의료법|상표·IP|기타",
      "requesterDept": "요청부서명",
      "date": "자문완료일"
    }
  ]
}
```

### 4.5 API 명세

| Method | Path | 설명 | 요청 | 응답 |
|---|---|---|---|---|
| `POST` | `/api/search` | AI 기반 FAQ 검색 | `{ query: string }` | FAQ JSON (totalFound, summary, faqs[]) |
| `GET` | `/api/stats` | DB 로드 상태 및 건수 확인 | — | `{ total, model, status }` |
| `POST` | `/api/reload` | Excel 파일 재로드 (데이터 갱신) | — | `{ total }` |
