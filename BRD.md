## 1. 배경 및 문제 정의

### 1.1 현황
재단법인 씨젠의료재단 준법감사팀은 사내 부서로부터 계약서 검토, 임대차, 노동·징계, 개인정보, 의료법, 공정거래 등 다양한 유형의 법률자문 요청을 지속적으로 처리해 왔다. 해당 법률자문 결과는 사내 협업 플랫폼(Seegene Feed)에 누적되어 있으나, 별도 검색 수단이 없어 과거 자문 사례를 실무에 재활용하기 어려운 상황이다.

### 1.2 핵심 문제

1. **자문 이력 재활용 불가**: 준법감사팀이 처리한 수십 건의 완료된 법률자문이 시스템에 축적되어 있으나, 이를 키워드로 검색하거나 유사 사례를 찾는 기능이 없어 동일·유사 질문이 반복 접수됨
2. **실무 부서의 자가 해결 불가**: 부서 담당자가 스스로 선례를 찾아 참고하고 싶어도, 준법감사팀에 직접 문의하거나 과거 이메일을 뒤지는 방법 외에 대안이 없음
3. **준법감사팀의 반복 응답 부담**: 이미 답변된 질문을 다시 받아 처리하는 비효율이 발생하며, 고도의 판단이 필요한 신규 업무에 집중하기 어려움
4. **지식 자산의 사장**: 완료된 자문 건은 담당자 개인이나 특정 시스템 내에 고립되어, 조직 차원의 법률 지식으로 체계화되지 못함

### 1.3 개선 필요성
축적된 법률자문 이력을 AI 기반 검색 시스템으로 연결하여, 실무 부서 담당자가 스스로 유사 사례를 조회하고 1차 참고 의견을 얻을 수 있는 환경을 구축한다. 이를 통해 준법감사팀은 신규·복잡 자문에 역량을 집중할 수 있다.

## 4. 비즈니스 요구사항

### 4.1 핵심 요구사항

**BR-01. 실제 자문 사례 기반 검색**
시스템은 준법감사팀이 처리·완료한 실제 법률자문 사례를 데이터로 사용해야 한다. 가상 사례나 외부 법령 DB가 아닌, 씨젠의료재단의 실제 맥락이 반영된 자문 이력에서 검색 결과를 제공해야 한다.

**BR-02. 키워드 기반 자연어 검색**
사용자는 법률 전문 용어를 몰라도 일상 업무 언어로 검색할 수 있어야 한다. AI가 키워드와 자문 사례를 의미론적으로 연결하여 관련 FAQ를 제시해야 한다.

**BR-03. FAQ 형식의 결과 제공**
검색 결과는 "질문–답변" 형식으로 구조화하여, 사용자가 자문 내용을 빠르게 파악할 수 있어야 한다. 답변에는 자문 완료일, 요청 부서, 법률 카테고리 정보가 포함되어야 한다.

**BR-04. 자가 서비스(Self-service) 구조**
실무 부서 담당자가 준법감사팀에 별도 요청 없이, 24시간 언제든지 스스로 검색하고 결과를 확인할 수 있어야 한다.

**BR-05. 데이터 보안 및 접근 통제**
자문 이력에는 부서명, 계약 상대방 등 민감 정보가 포함된다. 외부에 무단 공개되지 않도록 접근 제한 및 데이터 관리 기준을 준수해야 한다.

**BR-06. 데이터 최신화 가능 구조**
신규 법률자문이 완료될 때마다 데이터베이스를 업데이트할 수 있어야 한다. 시스템 중단 없이 데이터 갱신이 가능해야 한다.

### 4.2 비기능 요구사항

| 항목 | 요구 수준 |
|---|---|
| 응답 시간 | 검색 결과 제공까지 60초 이내 |
| 데이터 범위 | 완료 상태(REVIEW_COMPLETED)이고 본문 댓글이 아닌 자문 건 전체 |

## 5. 제약 사항 및 가정

### 5.1 제약 사항
- AI 검색 결과는 **참고용 1차 의견**이며, 법적 최종 판단이 아님
- 검토 의견의 법적 해석은 반드시 **준법감사팀의 확인**을 거쳐야 함
- 현재 한국어 자문 사례만 지원
- Excel 원본 파일의 형식(`t_feed(본문내용)` 시트, 지정 컬럼)이 변경되면 재배포 필요
- 자문 사례 데이터는 배포 시점 기준으로 고정되며, 갱신 시 재배포(또는 `/api/reload` 호출) 필요

### 5.2 가정
- 사내 환경에서 Google Gemini API 호출이 가능
- Excel 파일의 `REVIEW_CONTENTS` 컬럼에 실질적 검토 내용이 10자 이상 기재되어 있음
- 준법감사팀이 검색 결과 품질을 주기적으로 모니터링하고 프롬프트를 개선함
